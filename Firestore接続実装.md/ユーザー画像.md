目的:
ヒロメルの /post で、ユーザーが画像ファイルを選んで投稿できるようにしたい。
今は imageUrl の手入力が前提なので、一般ユーザーが使いにくい。
今回は Firebase Storage を使って、最小構成の画像アップロード機能を追加したい。

最重要条件:
今回の目的は「画像アップロード追加」だけです。
既に動いている Auth / キーパス / HomeButton / 投稿機能を壊さないことを最優先にしてください。

絶対に守ること:
- Googleログイン機能を壊さない
- キーパス制御を壊さない
- AuthProvider / AuthButtons / layout.tsx を不要に触らない
- HomeButton を触らない
- UI大改修をしない
- 複雑な新設計を入れない

前提:
- Next.js App Router
- Firebase Hosting
- Firestore 利用中
- Firebase Storage は利用可能
- /post は現在、ログイン + キーパス通過者のみ投稿可能
- ads コレクションには imageUrl フィールドがある
- 画像なし投稿も現状で可能

今回実装したいこと:
1. /post に画像ファイル選択UIを追加する
2. ユーザーが選んだ画像を Firebase Storage にアップロードする
3. アップロード後に取得したダウンロードURLを imageUrl に入れる
4. そのURLを使って既存の ads 投稿処理を行う
5. 画像なし投稿も引き続き可能にする

画像仕様:
- 1枚のみ
- 形式: JPEG / PNG / WebP
- 最大: 3MB
- 3MB超過または非対応形式はエラーメッセージ表示
- 画像は任意（なくても投稿可）

推奨するStorage構成:
- 保存先例: images/{uid}/{timestamp}-{filename}
- 必要なら安全なファイル名に整形
- contentType を使えるなら使う

推奨UI:
- 既存の /post フォームに以下を追加
  - ラベル: 画像（任意・3MBまで）
  - ファイル選択ボタン
  - 選択中ファイル名表示
- 余力があれば簡易プレビュー
- ただし見た目より安定優先
- 現在の imageUrl 手入力欄は以下のどちらかでよい
  1. 残す（保険）
  2. 非表示にする
  ただし、コードを壊さない方を優先する

エラーメッセージ例:
- 対応していない画像形式です
- 画像は3MB以下にしてください
- 画像アップロードに失敗しました
- 投稿に失敗しました

Firestore / Storage rules:
- 必要な最小変更のみ行う
- Storage は、認証済みユーザーのみアップロード可でよい
- できれば creatorAccess 承認済みユーザーのみアップロード可が理想
- ただし複雑になりすぎるなら、まずは認証済みユーザーのみでよい
- 既存の ads / clicks / creatorAccess rules を壊さない
- rules変更がある場合は最後に明記する

実装の進め方:
1. 現在の /post 投稿処理を読む
2. 変更範囲を /post と storage rules / firebase.ts の必要最小限に絞る
3. Authやキーパス周辺には不要な変更を入れない
4. まず画像アップロード成功を優先する
5. build を通す
6. 必要なら hosting / storage / firestore の deploy 手順を書く

完了条件:
- ログイン + キーパス通過済みユーザーが /post で画像を選択できる
- 画像を選んで投稿すると Storage に保存される
- 保存された画像URLで ads に投稿できる
- 投稿後、一覧・モーダルで画像が表示される
- 画像なし投稿も従来通り可能
- 3MB超過と非対応形式でエラーが出る
- 既存ログイン機能を壊していない
- 既存キーパス機能を壊していない
- build が通る
